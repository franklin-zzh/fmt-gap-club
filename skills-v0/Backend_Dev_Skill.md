# Skill_ID: Backend_Dev_Skill

## 1. 触发条件
由 `Agent.md` 在 [Step 2] 阶段并行唤醒，独立于前端运行。

## 2. 职责范围与工程输出
专注于高可用账户体系与高安全性支付状态机的构建，确保财务及生物数据零差错：

*   **轻量数据建模**：输出满足 MVP 要求的轻量级关系型数据库 Schema，包含：
    *   `User`（管理用户基本账户与认证信息）
    *   `Subscription`（管理会员资格状态、等级、起止有效期）
    *   `Order`（管理支付流水、第三方交易凭证、金额与状态变更日志）
*   **安全鉴权引擎**：编写标准的 JWT / OAuth 2.0 状态保持中间件，处理账户注册、登录与敏感接口的权限校验。
*   **异步 Webhook 幂等状态机**：编写处理第三方支付网关（Stripe/微信/支付宝）异步通知的核心逻辑。当接收到支付成功 Webhook 时，必须前置执行幂等性校验（防止因网络重试导致重复升级），校验通过后将用户订阅状态由 Pending 安全变更为 Active，并记录有效期。
